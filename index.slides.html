Using Neural Network to¶

Extract Feature from Job Descriptions¶

Problem¶

On the job market, there is a question always existing for the employer, which is how to hire a right person with a right salary? Actually, there is a similar question for job seekers, which is how to get a right job with right salary? The match point obviously is the salary! But under the salary, there are many factors influencing the match result, such as job location, working hours, company, job descriptions, etc..

Project Goal¶

To solve this problem with Machine Learning techniques, data scientists need to extract structured data from those factors, and use the structured data, also called features to predict the job salary.

In our project, we focus on the job descriptions of the job posts. Comparing with the job location, company, hours, the job description is much less structured. Manually identifying key meanings from mass job descriptions is a heavy job, so we want to find What 'Machine' has learned from our data.

We will use the word embedding vector method, and basic neural network model, to get trained sentences weights, or description-weights, as the representations of the descriptions, and see what kind of relationship existing between the extracted feature and the target variable which is the salary.

Finally, we will use some baseline models such as Random Forest Regressor, using the 'Category' variable created by the Adzuna company, and the description-weights, the feature we extracted, to separately predict salaries, and compare their performances. Even, if we are lucky, the relationship between the feature and the target variable, will tell me what kind of model is better for fitting.

Data Source¶

The data set comes from a Kaggle competition Job Salary Prediction.

This competition was launched five years ago, then we cannot join the competition and try to gain any rewards. But the data set of this competition is realistic, and is good for us to implement our project goal.

There are 244,768 data points in the training data set, and 122,463 samples in the test data set. In addition, there are other 122463 benchmark salary data points for calculating the Mean Absolute Errors.

Not only the Job description and the Job Category, there are other variables like Job Title, Company, Contract Type, Contract Time. In this project, we will focus on extracting a feature from the Job descriptions.

Data Preprocessing¶

Tokenize the texts of job descriptions¶

We use the Tokernizer of Keras to tokenize the cleaned contents of job descriptions. Keras' Tokenizer is a good utility to vectorize a text corpus. We use most all default arguments only assigning the num_words with the value of 167000 for covering words in our training data as much as possible. By default, Tokenizer will remove all punctuation, tabs and line breaks; and convert text to lower case.

In [36]:
max_words = 167000
tokenizer = Tokenizer(num_words=max_words)
tokenizer.fit_on_texts(desc_contexts)

Tokenizer can turn each job description into a sequence of integers (each integer being the index of a token in a dictionary). Then we can have a set of tokenized data points as data inputs, corresponding to job descriptions. To keep each input has the same length, we can use the pad_sequences of Keras to do that. We assign the maxlen argument of pad_sequences with the value of 512, which means that, if the number of tokens of one description is smaller than 512, then it will be padded with value zeros at the end, and if the number is bigger than 512, then it will be truncated to fit the desired length which is 512.

Once the tokenizer fitted the texts in the job descriptions, we can acquire the corpus via the attribute tokenizer.word_index, it's a dictionary, the key is each word in the corpus, the value is the corresponding index in the corpus. We also created a reverse dictionary, then we can use an index to inquire a word.

Create X_train, y_train, X_test, y_test from training dataset¶

We generated a set of random indices within the range of the number of all observations in the training dataset, and use it to set 190,000 samples as the training inputs - X_train, y_train , for our neural network model, and set the left 54768 samples as the testing inputs - X_test for prediction.

Word Embedddings¶

After tokenizing input data, then we will use the pre-trained Glove embeddings as the initiative weights, corresponding the tokenizer's corpus, to generate a 167000 X 100 matrix as the weights of the later neural network embedding layer.

The Glove cannot cover all words appeared in our tokenizer's corpus, there are 112,092 words will have initiated values with zeros. But don't worry, we just want to see how a neural network adjusts the weights matrix fitting our target salary values.

current_path = os.getcwd()

glove_embeddings_dict = get_glove_embeddings(glove_file)

embedding_dim = 100
desc_embeddings = np.zeros((max_words,embedding_dim))
out_glove_words = {}
for word, i in word_idx.items():
    if i < max_words:
        word_vector = glove_embeddings_dict.get(word)
        if word_vector is not None:
            desc_embeddings[i] = word_vector
        else:
            out_glove_words[word] = i

Training and Prediction¶

Using simple neural network model to train word embedding vectors¶

We use a Keras Sequential model, adding an Embedding layer as the first layer, and set it's weights with the matrix we have generated before. Next, we add a Flatten layer, three Dense layers with 100 units, and one Dense layer with 50 units. The activations of the four Dense layers are both the 'relu' function. At last, we add a Dense layer with 1 unit as the final output. The output will be the salary we want to predict, according the tokenized sequence we input.

In [44]:
model = Sequential()
model.add(Embedding(max_words,embedding_dim,input_length=seq_len))
model.add(Flatten())
model.add(Dense(100, activation='relu'))
model.add(Dense(100, activation='relu'))
model.add(Dense(100, activation='relu'))
model.add(Dense(50, activation='relu'))
model.add(Dense(1))
model.layers[0].set_weights([desc_embeddings])
model.compile(loss='mean_squared_error', optimizer='adam')

Then we train this model with the inputs X_train and y_train, we set the batch_size to 128, epochs to 20, and validation_split to 0.2. The validation_split argument brings a great functionality. In our case, it tells the training process to split 20% input samples for validation, somehow like the Cross-validation, but with a much less computing cost.

Adventure of finding pattern¶

In fact, the thing that the Machine has learned from our data, is exactly just the embedding layer weights in our trained Sequential model.

Now we can see, among the predicted salaries, their related job description contents, and the corresponding word weights, if there is any existing pattern.

Full Observations study via sequence representation¶

Let's make a scatter plot with all samples in the test data. Each data point on the plot with two properties: sequence weight, predicted salary.

One sequence means one specific description. We can extract all words first dimension weights in a sequence from the embedding layer, and use the average weight as the sequence representation. Using average weight is a simple way, but it can be the baseline and easy for finding patterns.

We will use the predicted salaries array as X values, and the sequence weights array as Y values.

Testing the new feature¶

Building a baseline RF model using 'Category' as X input¶

We firstly just use the 'Category' as the only feature. We use Sklearn LabelEncoder to encode the 'Category' variable as the X input.

Then we build a model with Sklearn RandomForestRegressor, training it with the encoded 'Category' values and the corresponding salary values, making a prediction with samples in the test data set. At last, we calculate the MAE between the predicted values and the benchmark values offered by the competition organizer.

Exponential function relationship?¶

The relationship between the sequence weights and the predicted values is more convergent than that with benchmark values. But both of the two relationships are nonlinear negative. When we treat salary variable as Y, and the sequence weights as X, then it will look like some sort of exponential function graph.

Discussion and further work¶

Let's review what we have done.

  1. We used a simple neural network to train the embedding layer weights via predicting target salary variable.
  2. And we use a few samples, which are beyond different salary levels, to find hints from the words and the corresponding weights.
  3. Then we create sequence representation using average word weight.
  4. We made a scatter plot between sequence weights and predicted salary values, finding the non-linear negative relationship.
  5. We use the sequence weights as a new feature applied on a simple RandomForestRegressor model, comparing to use the 'Category' variable on a very similar model.
  6. Finally, we applied the new feature to a linear model with the transformation of the y target variable. It achieved an impressive improvement on MAE score!

Through the experiment in this project, we found that there is a possible path using a neural network to learn and find patterns, or knowledges from data. Not just machine supervised by human, the two sides, human and machine, can teach each other and learn from each other.
